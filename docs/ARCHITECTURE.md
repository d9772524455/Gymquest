# Gym Quest — Отчёт об очистке кодовой базы

Gym Quest — геймифицированная платформа для фитнес-клубов: веб-приложение для атлетов, кабинет владельца клуба и мобильное приложение.
В рамках работы по очистке кодовой базы с 22 по 23 апреля 2026 года закрыто 50+ багов, архитектура приведена к production-стандарту.

---

## Сервер (Фаза 1)

Монолит в 515 строк разбили на 18 файлов по отдельным доменам: клубы, атлеты, тренировки, рейтинг, QR, достижения.
Добавили систему миграций — теперь изменения схемы базы данных накатываются безопасно, без ручного SQL.
Ввели валидацию всех входных данных: пароли (мин. 8 символов), email (формат + нормализация), диапазоны числовых полей.
Сервер теперь не запускается, если критичные настройки (JWT-секрет, порт БД) не заданы — ошибка сразу, не в рантайме.
Логи переведены в структурированный JSON-формат для удобства диагностики.
**Закрыто 16 багов безопасности и корректности.**

## Веб-приложение и кабинет клуба (Фаза 2)

Монолитные HTML-файлы (каждый по 200–270 строк встроенного кода) разделены на отдельные модули CSS и JavaScript.
Устранены **две XSS-уязвимости**: имена атлетов в списке участников и рейтинге больше не могут содержать вредоносный код.
QR-генерация на дашборде перенесена с внешнего сервиса `api.qrserver.com` (который видел токены клуба) на локальную библиотеку — токен больше никуда не уходит.
Добавлен автоматический выход при истёкшей сессии — вместо пустого экрана пользователь видит форму входа.
Дата окончания сезона теперь не может быть раньше даты начала — ни в форме, ни на сервере.
**Закрыто 14 багов.**

## Мобильное приложение (Фаза 3)

Android APK очищен от мёртвого кода: нерабочая геолокация и уведомления (FCM не настроен) удалены вместе с лишними разрешениями — приложение больше не запрашивает доступ к местоположению при первом запуске.
Устранена уязвимость QR-сканера: содержимое отсканированного QR-кода теперь безопасно экранируется перед передачей в приложение, вредоносный QR не может выполнить произвольный код.
Кнопка «назад» на главном экране теперь корректно закрывает приложение.
Собран **APK v5**, передан клиенту.
**Закрыто 8 багов.**

## Тесты и CI/CD (Фаза 4а)

Написано **35 автоматических тестов** на бизнес-логику: начисление XP, расчёт уровней, серии тренировок, георасчёты, валидация данных.
Настроен GitHub Actions: при каждом изменении кода автоматически запускаются проверка стиля, типов, unit-тесты и smoke-тест против временной базы данных Postgres.
Ничего сломанного не попадёт в продакшн — CI не пропустит.
**Закрыто 8 инфраструктурных багов.**

## Что не вошло в объём работ

Для прозрачности — что было обсуждено, но не сделано в рамках этого заказа:

- **Timezone-корректный подсчёт серий** — требует изменения схемы базы данных; отдельный заказ.
- **Переезд JWT на httpOnly cookies** — источники XSS закрыты, но для максимальной безопасности нужна переделка системы авторизации; отдельный заказ.
- **iOS / App Store** — нет Apple Developer аккаунта (~$99/год).
- **Google Play Console** — вы отказались; APK раздаём напрямую по ссылке.
- **SMTP email-алерты** — креды настроены, но reg.ru Cloud блокирует исходящие SMTP-порты; нужно открыть тикет в поддержку reg.ru на разблокировку.
- **FCM push-уведомления** — требуют настройки Firebase и отдельного файла конфигурации Android.
